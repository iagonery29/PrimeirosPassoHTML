Surgimento da Web

A web, ou World Wide Web, surgiu no final dos anos 1980 e início dos anos 1990, como uma forma de facilitar o compartilhamento de informações entre cientistas e pesquisadores ao redor do mundo. Ela foi criada por Tim Berners-Lee, um cientista britânico da área de computação, enquanto ele trabalhava no CERN (Organização Europeia para a Pesquisa Nuclear) em Genebra, Suíça.

Em 1989, Berners-Lee propôs um sistema baseado na tecnologia de hipertexto que permitiria a qualquer pessoa acessar documentos e informações por meio de links em uma rede. Esse sistema foi projetado para ajudar pesquisadores a compartilhar dados de forma mais rápida e eficiente. Em 1991, ele lançou o primeiro site da web e a primeira versão do navegador web, chamado WorldWideWeb (sem espaços, bem no estilo da época).

A relação clients e servers

A relação entre clients(clientes) e servers(servidores) é um conceito fundamental na arquitetura da web e das redes de computadores em geral. Ela pode ser explicada assim:

1. Client (Cliente)

O cliente é o dispositivo ou software que solicita recursos ou serviços de um servidor. Pode ser qualquer coisa, desde um navegador de internet (como o Chrome, Firefox ou Safari) até um aplicativo no celular ou no computador. O cliente é responsável por fazer requisições para o servidor e exibir ou processar os dados que ele recebe.
Exemplo: Quando você acessa um site na web, seu navegador atua como um cliente. Ele faz uma solicitação ao servidor onde o site está hospedado, pedindo para que ele envie a página para ser exibida na tela.

2. Server (Servidor)

O servidor é o dispositivo ou software que fornece recursos, dados ou serviços aos clientes que fazem solicitações. Ele responde às requisições do cliente, processando-as e devolvendo os resultados solicitados. Os servidores geralmente estão configurados para atender a múltiplos clientes ao mesmo tempo, fornecendo serviços como hospedagem de sites, e-mail, compartilhamento de arquivos, etc.
Exemplo: Quando você digita uma URL no navegador (como "www.exemplo.com"), você está fazendo uma solicitação a um servidor que hospeda aquele site. O servidor, então, envia de volta os arquivos necessários para renderizar a página que você vê no seu navegador.

3. Como funciona a comunicação
  1. Requisição: O cliente (como o navegador web) envia uma requisição para o servidor. No caso da web, essa requisição geralmente é feita usando o protocolo HTTP (HyperText Transfer Protocol), que define como os dados são transmitidos.
  2. Processamento no Servidor: O servidor recebe essa requisição e processa o pedido. Se for um site, por exemplo, ele pode buscar os arquivos da página, imagens ou outros recursos no seu sistema de arquivos, ou então consultar um banco de dados, dependendo da complexidade do pedido.
  3. Resposta: Após o processamento, o servidor envia uma resposta ao cliente, que pode incluir os dados solicitados, como o código HTML de uma página, imagens ou até mesmo mensagens de erro, caso haja algum problema (por exemplo, "404 - Página não encontrada").
  4. Exibição: O cliente recebe esses dados e os exibe de forma adequada para o usuário, como a página web que você vê no navegador.

Tags HTML

  • h1 a h6: Representam títulos de diferentes níveis (h1 é o mais importante e h6 o menos importante).
  • p: Usada para criar um parágrafo de texto.
  • br: Quebra de linha (sem tag de fechamento).
  • strong: Torna o texto em negrito, indicando ênfase.
  • i ou em: Torna o texto em itálico, indicando ênfase.
  • a: Usada para criar links.
  • img: Insere uma imagem.
  • ol: Listas Ordenadas (numeradas).
  • ul: Listas Não Ordenadas (com marcadores).
  • li: Usada para cada item da lista (ordenada ou não ordenada).
  • !-- --: Usada para fazer comentários, coloca-se o texto no espaço em branco.
